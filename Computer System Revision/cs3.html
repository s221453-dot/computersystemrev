Software & Operating Systems

1. System Software vs. Application Software

Type Purpose Examples
System Software Manages hardware and provides a platform for applications. Operating Systems, Utilities, Device Drivers.
Application Software Performs specific tasks for the end-user. Word processors, Games, Web browsers.

2. The Operating System (OS)

Core Functions of an OS

  • Resource Management: Allocates CPU time and Memory to different tasks.
  • File Management: Organizing, naming, and storing files on storage devices.
  • User Interface: Allowing the user to communicate with the computer.
  • I/O Control: Managing communication with printers, keyboards, etc.

GUI vs. CLI

  • GUI (Graphical User Interface): Uses icons, windows, and a mouse. Pro: User-friendly. Con: Uses more system resources.
  • CLI (Command Line Interface): Uses text-based commands (e.g., cmd). Pro: Fast for experts; low resources. Con: Hard to learn commands.

System Utilities & Drivers

  • Device Driver: Small program that tells the OS how to communicate with a specific piece of hardware (e.g., a new printer).
  • Disk Defragmentation: A utility that rearranges files on a Hard Disk so they are stored in contiguous blocks. This speeds up file access. (Note: Not needed for SSDs!)
